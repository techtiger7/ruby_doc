Introduction

The aim of this documentation is to provide simple instructions for an intermediate to experienced programmer to get started with Ruby, it is not for beginner programmers, but more for those coming from other languages who wish to learn Ruby and don't want to trawl through all the basic information describing "what is a variable" and "how do I write a function?". This doesn't explain how to set up your Ruby environment, just details on the syntax so you have enough to get started writing code, the rest is on you.

As an interpreted language (like JavaScript) Ruby code executed at run time, straight from the instructions within our code, this is important to remember as it effects the manner in which we write code, and the efficiency of Ruby.

If there is information missing within these instructions, or something is incorrect, please let me know, I have developed this myself while learning Ruby coming from Java, so don't use this as a definitive guide, just a starting point and seek further clarification from Stackoverflow, Tutorialspoint or any of your other favourite web resources.

Printing

When we want to print something to standard output in Ruby, we have two main options available to us, puts which will 'put' the passed in string value to standard output (this can also be attached to an Object like a File instance as a method for writing to it, but this is more advanced usage.) This string will be followed by a line return character, so any further content will be written to the next line.

Alternatively we can use the print keyword, which will 'print' the passed in value to standard output, but will not include a line return, so if we 'print' two things one after the other, they will appear on the same line.

Puts example:

              puts "Hi, I'm Tom."
              puts "I love Ruby."
              # Hi, I'm Tom.
              # I love Ruby.

Print example:

              print "Hi, I'm Tom."
              print "I love Ruby."
              # Hi, I'm Tom.I love Ruby.

Storing Input

Another key requirement for any programming language is user inputs, these are enabled in Ruby by the gets keyword, which reads from standard input and can write this to a variable.

Gets example:

              puts "Please enter your first name:"
              gets first_name
              puts "Hi there " + first_name + "!"
              
              # Please enter your first name:
              # Tom    # User inputs "Tom" and presses Enter
              # Hi there Tom!

Variables

Like most other programming languages, Ruby lets you define variables for storing data. There are four main types of variables, each with varying scope levels, these are:

  • Local variable scoped to the method or class it is defined in. eg. my_var
  • Instance variable scoped to this entire class instance, any method in this instance can access this. eg. @my_var
  • Class variable scoped to every instance of this class. eg. @@my_var
  • Global variable scoped to everything. eg. $my_var
  • Note: You can access any variable using a hashtag, for example #my_var

Classes

Ruby is an Object Oriented programming language, with Classes which are defined based on the keyword class and a Capital case label.

For example:

              class Student
                Body of class
              end

As opposed to using curly brackets as in Java or other languages, Ruby indicates the termination of a block of code with the keyword end.

Methods

Methods define the behaviours of classes in Ruby, in the same manner they do for other OOP languages, in order to define these methods in a class we utilise the def keyword, followed by a label for the name of the method.

For example:

              class Student
                @@number_of_students = 0
              
                def count_students
                  puts number_of_students
                end
              end

Once again, Ruby indicates the termination of a block of code with the keyword end.

Class Instances

To define an instance of a class (or to create an object of particular class) we utilise the new keyword.

For example:

  student1 = Student.new

This creates a local variable named 'student1' which references a new instance Object of the class 'Student'.

Initializations

Class constructors in Ruby are referred to as initializations and need to be explicitly declared within the class by creating a method with the label of initialize so that new class instance parameters are defined for set up of an object.

For example:

                class Student
                  @@number_of_students
                
                  def initialize(first_name, last_name, student_id)
                    @first_name = first_name
                    @last_name = last_name
                    @student_id = student_id
                  end
                end

Inheritance

Class inheritance is controlled in Ruby by the < symbol.

For example:

                class Dog
                  @legs = 4
                end
                class Poodle < Dog
                curly = Poodle.new
                curly.instance_variable_get("@legs")

The instance of the Poodle class referenced by the local variable curly will return the value of 4 for @legs, because this has been inherited from the class Dog where @legs is already 4. The same applies to methods, these will be inherited too, and can also be overridden if we define them again within the class we're creating.

Super

When a class is inherited from another class, it has all the same methods, these can be overwritten through the definition of a new method with the same name. Alternatively, if we want to extend the existing method, we can do so through use of the super keyword, which returns the content of same method in the the superclass (parent class from which this class inherited its methods).

For example:

                class Dog
                  def initialize(no_of_legs)
                    @no_of_legs = no_of_legs
                  end
                end
                
                class Poodle < Dog
                  def initialize(no_of_legs, is_minature)
                    super
                    @is_minature = is_minature
                  end
                end

Here the superclass of Dogs has an initialize method which only takes one parameter no_of_legs to store the number of legs the dog instance has, but we have overridden this on the inherited class of Poodle, whereby we added the parameter of is_minature to the Poodle class, so we can store whether each Poodle instance is a minature poodle or not.

By using the super keyword, access the code within the initialize method of the superclass Dog, hence we only have to store one instance variable of @is_minature as the other parameter is handled by the inheritance.

Public, Private and Protected

Similar to other OOP languages, Ruby has various protection levels for methods in order to prevent access outside of a class or object, the three levels available are:

  • public
  • protected
  • private

Unlike other languages, you do not need to set the accessibility of instance variables, this happens implicitly, with all instance variables being private, and all methods being public. To change this, access to these variables needs to be dictated through the public, private and protected keywords.

Public methods   student1 = Student.new

This creates a local variable named 'student1' which references a new instance Object of the class 'Student'.
